# LeaderBoard Marketplace: Backend Infrastructure

Цей репозиторій містить інфраструктуру бази даних для SaaS-платформи маркетплейсу, побудовану на базі **Couchbase Server**. Проєкт використовує Docker для швидкого розгортання та автоматизовані скрипти для ініціалізації структури даних.

## 🚀 Швидкий старт (з нуля)

Дотримуйтесь цих кроків, щоб запустити локальне середовище розробки.

### 1. Попередні вимоги
Переконайтеся, що у вас встановлені:
* **Docker Desktop** (або Docker Engine з Docker Compose)
* **Bash** (для запуску скриптів ініціалізації)

### 2. Розгортання контейнера
Запустіть Couchbase Server за допомогою Docker Compose:
```bash
docker-compose up -d

### 3. Ініціалізація структури та даних

Після успішного старту контейнера необхідно налаштувати логічну структуру бази (Buckets, Scopes, Collections) та завантажити початкові дані з папки `/data`.

**Порядок дій:**

1. **Надайте права на виконання скрипту:**
   Відкрийте термінал у корені проєкту та виконайте:
   ```bash
   chmod +x ./setup_collections.sh

2. **Запустіть скрипт ініціалізації:**
	./setup_collections.sh


## 🧩 Сервіс API (Backend)

Сервіс побудований на базі **Node.js** та розроблений для обробки запитів клієнтської частини, керування ролями користувачів та виконання складних операцій з базою даних.

### 🛠 Технології API:
* **Node.js & TypeScript** — середовище виконання та типізація.
* **Couchbase SDK** — офіційний драйвер для високоефективного зв'язку з БД.
* **Express/NestJS** — фреймворк для побудови RESTful API.

### 📋 Функціональні можливості:
1. **Role Switcher Logic:** Обробка запитів залежно від поточної ролі (Admin, Seller, Buyer) без повноцінної авторизації.
2. **Sub-Document API:** Оптимізоване отримання окремих полів документів (ціни, залишку) для миттєвого оновлення інтерфейсу.
3. **ACID Transactions:** Надійне оформлення замовлень, що гарантує цілісність даних у трьох колекціях одночасно.
4. **FTS Integration:** Проксі-запити до вбудованого пошукового сервісу Couchbase.
